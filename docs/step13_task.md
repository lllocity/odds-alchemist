# タスク: Step 13 定期スクレイピングの自動化

## 目的
Spring Bootのスケジューリング機能を利用し、指定した対象レースのオッズ情報を定期的に自動取得・保存する仕組みを構築する。

## 実行要件
1. **スケジューラーの有効化**:
   Spring Bootアプリケーション設定クラス（メインクラス等）で、定期実行を有効にする設定（`@EnableScheduling`）を追加する。
2. **スケジューラークラスの実装**:
   `OddsScrapingScheduler` 等のクラスを新規作成し、定期的に実行される処理を実装する。
3. **設定の外部化**:
   実行間隔（Cron式や固定ディレイ時間）および、監視対象のスポナビ競馬URLはソースコードにハードコードせず、`application.yml` から読み込む設計とすること。
4. **処理の連携とロギング**:
   定期実行メソッド内から `OddsSyncService` を呼び出し、処理の開始と完了、またはエラー発生時のスタックトレースを適切なログレベル（INFO / ERROR）で出力すること。
