# タスク: Step 15 異常検知アラートのフロントエンド表示

## 目的
バックエンドが検知した「オッズの異常（人気の盲点）」を、Next.jsのフロントエンドでリアルタイムに確認できるUIを構築する。

## 実行要件
1. **バックエンド (Spring Boot) - APIの実装**:
   - `OddsAlertController` を作成し、最新のアラート一覧を返すGETエンドポイント（例: `/api/odds/alerts`）を実装する。
   - ローカルのNext.js（`localhost:3000`等）からのリクエストを許可するCORS設定を適切に行う。

2. **フロントエンド (Next.js) - データ取得処理**:
   - APIエンドポイントからデータを取得し、常に最新状態を保つためのポーリング処理（`useEffect` + `setInterval`、またはSWR等）を実装する。
   - バックエンドのDTOと一致するTypeScriptの型定義（インターフェース）を正確に作成する。

3. **フロントエンド - UIコンポーネントの実装**:
   - 検知されたアラートを一覧表示するコンポーネントを作成する。
   - 表示項目: 馬番、馬名、検知タイプ（支持率急増、単複順位乖離、トレンド逸脱など）、検知時刻、該当数値。
   - スタイリング: Tailwind CSSを使用し、過度な装飾を抑えたフラットで視認性の高いデザインにする。検知タイプによってラベルの色を分けるなど、直感的に状況を把握できるようにする。

4. **【重要】ドキュメントの自律的ブラッシュアップ**:
   - 本タスクの実装完了後、今回の「Next.jsとSpring Boot間のCORS連携」や「TypeScriptの型定義ルールのベストプラクティス」について、今後もシステム全体で遵守すべきルールを `CLAUDE.md` または `docs/skills.md` に自ら追記・整理してコミットに含めること。
