# タスク: Step 16 検知機能の拡張と運用フローの高度化

## 目的
ユーザーインターフェースの改善（検知種別の解説、レース名表示、URL動的登録）と、検知されたアラートデータのGoogle Sheetsへの永続化を実装する。

## 実行要件

1. **フロントエンド: 検知種別のヘルプ/説明表示**
   - アラート一覧画面（`AlertDashboard`等）の上部またはサイドに、検知種別（「支持率急増」「単複順位乖離」「トレンド逸脱」など）のロジックや意味を簡潔に説明する凡例（レジェンド）またはツールチップを追加する。

2. **バックエンド＆フロントエンド: レース名の取得と表示**
   - `RaceOddsParser` (Jsoup) を改修し、対象URLのHTMLから「レース名（例: 第91回 日本ダービー）」をスクレイピングして抽出する。
   - `AnomalyAlertDto` に `raceName` フィールドを追加し、フロントエンドのアラート一覧の各行（またはカード）にレース名を表示する。

3. **監視対象URLの動的登録（UIからの設定）**
   - **フロントエンド**: 監視対象のスポナビURLを入力してバックエンドに送信するためのシンプルな入力フォーム（Inputと登録Button）を作成する。
   - **バックエンド**: URLを受け取るPOSTエンドポイント（例: `/api/odds/targets`）を実装し、受け取ったURLリストをスレッドセーフなインメモリ（`CopyOnWriteArrayList` や `ConcurrentHashMap` のKey等）で保持する。
   - **スケジューラーの改修**: `application.yml` からURLを読み込む仕様を廃止し、インメモリに登録されたURLリストをループして、それぞれのオッズを取得・判定する仕様に変更する。

4. **検知データ（アラート）のスプレッドシート永続化**
   - 既存のGoogle Sheets連携処理（`OddsSyncService` 等）を拡張し、異常が検知されたタイミングで、アラート専用のシート（例: "Alerts" タブ）へデータを書き込む処理を追加する。
   - 書き込み項目: 検知日時、対象URL、レース名、馬番、馬名、検知タイプ、該当数値。
   - ※読み込みは行わず、書き込み（Append）のみの一方向の永続化とする。
