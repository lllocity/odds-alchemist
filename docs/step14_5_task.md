# タスク: Step 14.5 異常検知ロジックの高度化（動的サンプリングと基準値比較）

## 目的
スクレイピングの実行間隔をレース発走時刻に応じて動的に変更する仕組みと、初期基準値（朝イチ）からのトレンド逸脱を検知するロジックを追加し、分析エンジンの精度を向上させる。

## 実行要件

1. **スケジューラーの動的変更 (`OddsScrapingScheduler` の改修)**:
   - 既存の固定間隔（`@Scheduled(fixedDelay = ...)`等）を廃止し、Springの `SchedulingConfigurer` や `TaskScheduler` とカスタム `Trigger` を用いて、実行間隔を動的に変更できるアーキテクチャにリファクタリングする。
   - スポナビのページから「レース発走時刻（締め切り時間）」を取得してパースする。
   - 現在時刻から発走時刻までの「残り時間」に応じて、次回の実行間隔を以下のように動的に切り替える。
     - 朝〜12:00（プール金・少）: 30分間隔
     - 12:00〜60分前（プール金・中）: 15分間隔
     - 60分前〜10分前（プール金・大）: 5分間隔（コアタイム）
     - 10分前〜直前（プール金・最大）: 1分間隔

2. **朝イチ基準値からのトレンド逸脱検知 (`OddsAnomalyDetector` の改修)**:
   - その日の初回取得データ（または12:00以前に取得した最も古いデータ）を「初期基準値」としてインメモリに永続化する。
   - 最新のオッズとこの「初期基準値」を比較する新たな検知ロジック（検知タイプ: `トレンド逸脱`）を実装する。
   - 特に中穴帯（5〜8番人気）および大穴（9〜12番人気）の馬において、初期基準値から支持率（単勝オッズの逆数）が一定の閾値以上上昇している場合、大口の先行投資シグナルとしてアラートを生成する。
